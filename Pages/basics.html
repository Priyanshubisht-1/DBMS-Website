Block Structure

PL/SQL programs are divided and written in logical blocks of code. Each block consists of:

  • Declaration Section: This section starts with the keyword DECLARE. It is an optional section and defines all variables, cursors, subprograms, and other elements to be used in the program.

  • Executable Commands: This section is enclosed between the keywords BEGIN and END. It is a mandatory section. It consists of the executable PL/SQL statements of the program.

  • Exception Handling : This section starts with the keyword EXCEPTION. This optional section contains exception(s) that handle errors in the program.

Syntax:

DECLARE 
   -- declarations section 
BEGIN 
   -- executable command(s)
EXCEPTION 
   -- exception handling 
END;

Example:

DECLARE 
   message  varchar2(20):= 'Hello, World!'; 
BEGIN 
   dbms_output.put_line(message); 
END; 
/

Output:

Hello, World!

Datatypes

PL/SQL supports various data types similar to SQL but with additional types for better control and flexibility. These types define the kind of data a variable can store, such as numbers, characters, dates, and more.

  • NUMBER(p, s): used to store numeric values.(p : Total number of digits, s : Number of digits to the right of the decimal)

  • VARCHAR2(n): Variable-length character string (n: size of string)

  • CHAR(n): Fixed-length character string (n: size of string)

  • DATE: stores date ('DD-MON-YY')

  • BOOLEAN: Stores TRUE, FALSE, or NULL

Example:

DECLARE 
    x number(5,3) := 29.111;
    y varchar2(20):= 'Hello, World!'; 
    z date := '01-APR-05';
BEGIN 
    dbms_output.put_line(x); 
    dbms_output.put_line(y); 
    dbms_output.put_line(z); 
END; 
/

Output:

29.111
Hello, World!
01-APR-05

Variable

A variable in PL/SQL is referred to as a storage location that holds a value of a certain data type. A variable must be declared in the declaration section of a block before being used.

Syntax:

variable_name datatype [NOT NULL] [:= initial_value]

Example :

DECLARE 
name varchar(20) := 'Priyanshu';
dob date:= '13-APR-05';
BEGIN 
dbms_output.put_line(name || ' was born in '|| dob);
END; 
/

Output :

Priyanshu was born in 13-APR-05

Constant

Constants are those values which when declared remain fixed throughout the PL/SQL block. For declaring constants, a constant keyword is used.

Syntax:

constant_name CONSTANT data_type := fixed_value;

Example :

DECLARE 
    name varchar(20) := 'Priyanshu';
    dob constant date:= '13-APR-05';
BEGIN 
    dob := '14-APR-05';	--ERROR
    dbms_output.put_line(name || ' was born in '|| dob);
END; 
/

Output :

ORA-06550: line 5, column 5:
PLS-00363: expression 'DOB' cannot be used as an assignment target

Variable Scope

PL/SQL allows the nesting of blocks. There are two types of variable scope:

  • Local variables: Variables declared in an inner block and not accessible to outer blocks.

  • Global variables: Variables declared in the outermost block or a package.

Example :

DECLARE 
    -- Global variables  
    num1 number := 95;  
    num2 number := 85;  
BEGIN  
    dbms_output.put_line('Outer Variable num1: ' || num1); 
    dbms_output.put_line('Outer Variable num2: ' || num2); 
    DECLARE  
      -- Local variables 
      num1 number := 195;  
      num2 number := 185;  
    BEGIN  
      dbms_output.put_line('Inner Variable num1: ' || num1); 
      dbms_output.put_line('Inner Variable num2: ' || num2); 
    END;  
END; 
/

Output :

Outer Variable num1: 95
Outer Variable num2: 85
Inner Variable num1: 195
Inner Variable num2: 185
